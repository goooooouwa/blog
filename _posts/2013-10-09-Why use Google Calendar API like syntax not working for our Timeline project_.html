--- layout: post title: 'Why use Google Calendar API like syntax not working for our Timeline project?' --- Why use Google Calendar API like syntax not working for our Timeline project?
Google Calendar API is like this:
POST https://www.googleapis.com/calendar/v3/calendars/primary/events?key={YOUR_API_KEY}

Content-Type:  application/json
Authorization:  Bearer ya29.AHES6ZQUNMWMkHUvmEuYStK_vcbfkFAozW2PhGiovkgJqrIKxg
X-JavaScript-User-Agent:  Google APIs Explorer

{
  "end": {
    "dateTime": "2013-01-23T10:00:00+08:00"
  },
  "start": {
    "dateTime": "2013-01-13T10:00:00+08:00"
  }
}

They do not have the redundant resource name within the json object( which is usually the case of Rails params hash object), which should look like this:
{
  "event":
  {
    "end": {
      "dateTime": "2013-01-23T10:00:00+08:00"
    },
    "start": {
      "dateTime": "2013-01-13T10:00:00+08:00"
    }
  }
}

So, why use Google Calendar API like syntax not working for our Timeline project?
Because we use Grape as API server. Grape will add route_info into the params hash object, like below( which is the content of a "Graped" params hash object):
#<Hashie::Mash category=\"c3\" route_info=#<Grape::Route:0x007fe687842108 @options={:description=>\"Create an event.\", :jbuilder=>\"event\", :prefix=>nil, :version=>\"v1\", :namespace=>\"/events\", :method=>\"POST\", :path=>\"/:version/events(.:format)\", :params=>{}, :compiled=>/\\A\\/(?<version>v1)\\/events(?:\\.(?<format>[^\\/.?]+))?\\Z/}> title=\"t3\">

Conclusion:
Some third party libraries often add extra info to the data transfered( e.g. model in Canjs, params in Grape), this causes the difficultly to create a simple API like the Google Calendar API. So it's safer to surround your parameter data explicitly in params hash. This can be seen as an example of how using third party library carelessly can cause problem.
