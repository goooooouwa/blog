--- layout: post title: 'asynchronous workflow is conceptually the same as synchronous workflow' categories: programming --- asynchronous workflow is conceptually the same as synchronous workflow
In synchronous workflow, the return statement indicates the end of a function's execution. And we can continue the workflow based on the return value.
In asynchronous workflow, the response of request indicates the end of a function's execution. And we can continue the workflow based on the response value. So we need callback functions to achieve this kind of workflow, return statement is not useful when managing the workflow in asynchronous environment.
