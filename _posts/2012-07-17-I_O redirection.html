--- layout: post title: 'I/O redirection' --- I/O redirection

A file descriptor is simply a number that the operating system assigns to an open file to keep track of it. Consider it a simplified type of file pointer. It is analogous to a file handle in C.

Each open file gets assigned a file descriptor. [2] The file descriptors for stdin, stdout, and stderr are 0, 1, and 2, respectively. For opening additional files, there remain descriptors 3 to 9. By default, both standard output and standard error are linked to the screen and not saved into a disk file. 

Example:

i>&j
      # Redirects file descriptor i to j.
      # All output of file pointed to by i gets sent to file pointed to by j.
      #把描述符i所指向的所有文件的输出重定向到j所指向的所有文件。
 
个人理解：标准输入，标准输出，标准错误输出，都是文件描述符，而键盘，屏幕是文件。1>&7 是将文件描述符1（标准输出）所指向的所有文件（屏幕）的输出重定向到文件描述符7所指向的所有文件。
